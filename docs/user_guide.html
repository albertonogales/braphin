User Guide

This guide walks through every stage of the BRAPHIN pipeline with complete, runnable code examples. By the end you will be able to go from a raw NIfTI file to a NetworkX functional-connectivity graph ready for graph neural network classification.

5-minute start

The Graph API runs the full pipeline in four lines:

from braphin import Graph

g = Graph()
g.load_data("sub-01_bold.nii.gz", modality="fmri")
graph, matrix = g.modelate(connectivity="pearson_correlation", window_size=None)
# matrix is a NumPy array (N_regions × N_regions) of connectivity weights
# graph  is a NetworkX graph where nodes are brain regions

That's it. The next sections explain what happens inside each stage and how to customise it.

Pipeline Overview

BRAPHIN processes fMRI data in five stages. Each stage reads the output of the previous stage, so you can inspect, save, or branch the data at any point.

Load file  →  Clean data  →  Remove noise  →  Parcellate into brain regions  →  Compute connectivity

In code, this maps to five classes:

InputfMRIData.load()                    →  BRAPHINInputBundle
    ↓
PreprocessBRAPHINData.run()             →  BRAPHINPreprocessBundle
    ↓
DenoiseBRAPHINData.run()                →  BRAPHINDenoiseBundle
    ↓
TransformBRAPHINData.run()              →  BRAPHINTransformBundle
    ↓
ModelBRAPHINConnectivityData.run()      →  BRAPHINConnectivityBundle

All classes are available from the top-level package:

from braphin import (
    InputfMRIData, PreprocessBRAPHINData, DenoiseBRAPHINData,
    TransformBRAPHINData, ModelBRAPHINConnectivityData,
    PreprocessConfig, DenoiseConfig, AtlasConfig, ConnectivityConfig,
)

Stage 1 — Loading Data

The first stage reads your fMRI scan file. NIfTI is the standard file format for brain scan data (.nii or .nii.gz). BRAPHIN validates the file and loads it along with any optional companion files (confounds, scan parameters).

Minimal example

from braphin import InputfMRIData

bundle = InputfMRIData("sub-01_task-rest_bold.nii.gz").load()

print(bundle.fmri_metadata["shape"])  # e.g. (91, 109, 91, 200) — x, y, z, time
print(bundle.fmri_metadata["tr"])     # repetition time in seconds, read from the file header

Loading with a confound file

Confound files contain measurements of noise sources — for example, head motion parameters recorded during the scan. Supported formats: .tsv, .csv, .npy, .json. 

bundle = InputfMRIData(
    "sub-01_task-rest_bold.nii.gz",
    auxiliary_paths=[
        "sub-01_task-rest_confounds_timeseries.tsv",
        "sub-01_task-rest_bold.json",    # optional scan-parameter sidecar
    ],
).load()

confounds_array = bundle.auxiliary_files["sub-01_task-rest_confounds_timeseries.tsv"]
print(confounds_array.shape)   # e.g. (200, 24) — 200 time points, 24 confound columns
BIDS confound files: BRAPHIN automatically detects and skips the text header row in BIDS-format confound files, so the returned array contains only numeric values.

TR (the time between consecutive brain scans, in seconds) is read automatically from the NIfTI file header. If you also load a BIDS JSON sidecar that contains a "RepetitionTime" field, that value takes precedence. When TR is found automatically, you do not need to set it manually in later pipeline stages.

Stage 2 — Preprocessing

Preprocessing prepares the raw scan data for analysis. Most users only need per-voxel normalisation, which rescales each brain location's signal to have a consistent range.

Minimal example — normalisation only

from braphin import PreprocessBRAPHINData, PreprocessConfig

pp_bundle = PreprocessBRAPHINData(
    input_bundle,
    PreprocessConfig(apply_voxel_zscore=True),
).run()

print(pp_bundle.applied_steps)    # ['per_voxel_temporal_normalisation']

All preprocessing steps

The following steps are available and always run in this fixed order when enabled:

  1. Invalid value removal — always on. Replaces any non-finite values with 0.
  2. Slice-timing correction — corrects for the fact that different brain slices are scanned at slightly different times. Requires TR.
  3. Motion correction — aligns every scan volume to the first one using a 6-parameter rigid-body fit (translations and rotations).
  4. Outlier detection — identifies scan volumes with abnormally large signal changes and interpolates them from their neighbours.
  5. Per-voxel temporal normalisation — z-scores each brain location's signal across time, so that all voxels have comparable scale.
  6. Spatial smoothing — applies a Gaussian blur across space to reduce noise.

Full example

pp_bundle = PreprocessBRAPHINData(
    input_bundle,
    PreprocessConfig(
        apply_slice_timing=True,      tr=2.0,            # TR in seconds
        apply_motion_correction=True,
        apply_outlier_detection=True,
        apply_voxel_zscore=True,
        apply_smoothing=True,         smoothing_fwhm=6.0,
    ),
).run()

print(pp_bundle.applied_steps)
# ['slice_timing_correction', 'motion_correction', 'outlier_detection',
#  'per_voxel_temporal_normalisation', 'spatial_smoothing']
print(pp_bundle.motion_params.shape)    # (200, 6) — [tx, ty, tz, rx, ry, rz]
print(pp_bundle.outlier_mask.sum())     # number of flagged volumes
Non-axial acquisitions: By default, BRAPHIN assumes slices run along the Z axis (axial acquisition). For coronal acquisitions, set slice_axis=1; for sagittal, set slice_axis=0. Example: PreprocessConfig(apply_slice_timing=True, tr=2.0, slice_axis=1).

Stage 3 — Denoising

Denoising removes signals from known noise sources — most commonly head movement — from the brain data. BRAPHIN fits a linear model to the nuisance signals and subtracts their influence from the brain data.

Confound regression

from braphin import DenoiseBRAPHINData, DenoiseConfig

dn_bundle = DenoiseBRAPHINData(
    pp_bundle,
    DenoiseConfig(regress_confounds=True),
).run()

print(dn_bundle.applied_steps)
# ['confound_regression']
print(dn_bundle.denoise_metadata["confounds_file_used"])
# 'sub-01_task-rest_confounds_timeseries.tsv'

BRAPHIN automatically finds the confound file you loaded in Stage 1 — it looks for the first auxiliary file whose number of rows matches the number of scan volumes.

Bandpass filtering

Bandpass filtering keeps only the signal frequencies associated with resting-state brain activity (typically 0.008–0.1 Hz) and discards slower drifts and faster noise. TR (in seconds) is required because the filter depends on the sampling rate. 

dn_bundle = DenoiseBRAPHINData(
    pp_bundle,
    DenoiseConfig(
        regress_confounds=True,
        apply_bandpass=True,
        tr=2.0,
        bandpass_low=0.008,
        bandpass_high=0.1,
    ),
).run()
print(dn_bundle.applied_steps)
# ['bandpass_filtering', 'confound_regression']
Filtering order: When both bandpass filtering and confound regression are enabled, BRAPHIN filters the confound signals with the same filter before regression. This prevents a spectral mismatch between the brain signal and the regressors that would otherwise inflate connectivity estimates (Lindquist et al. 2019).

Scrubbing

Scrubbing interpolates (or marks) scan volumes that have excessive motion. If outlier detection was run in Stage 2, the same flagged volumes are used here; otherwise BRAPHIN recomputes them from the denoised signal.

dn_bundle = DenoiseBRAPHINData(
    pp_bundle,
    DenoiseConfig(regress_confounds=True, apply_scrubbing=True),
).run()

Motion confound regression

After motion correction, use get_motion_confounds() to obtain the motion parameters ready for use as confound regressors.

from braphin import get_motion_confounds

motion_confounds = get_motion_confounds(pp_bundle)
print(motion_confounds.shape)   # e.g. (200, 6)

# Add to the auxiliary files so denoising picks them up automatically
pp_bundle.auxiliary_files["motion_confounds.npy"] = motion_confounds

Skipping denoising

dn_bundle = DenoiseBRAPHINData(
    pp_bundle,
    DenoiseConfig(regress_confounds=False, apply_scrubbing=False, apply_bandpass=False),
).run()
# applied_steps will be empty — data passes through unchanged

Stage 4 — Atlas Parcellation

An atlas is a map that divides the brain into named regions (ROIs — regions of interest). BRAPHIN assigns each brain voxel to a region and computes the average signal for that region over time. This reduces the data from tens of thousands of voxels to a manageable number of region signals.

Using a bundled atlas

from braphin import TransformBRAPHINData, AtlasConfig

tx_bundle = TransformBRAPHINData(
    dn_bundle,
    config=AtlasConfig(atlas_name="aal"),  # 116 brain regions
).run()

print(tx_bundle.roi_time_series.shape)     # (116, 200) — 116 regions × 200 time points
print(tx_bundle.roi_labels[:3])            # ['Precentral_L', 'Precentral_R', ...]

If the atlas and fMRI scan have different voxel sizes or dimensions, BRAPHIN automatically resamples the atlas to match the scan using nearest-neighbour interpolation, preserving region label integrity.

Using a custom atlas

tx_bundle = TransformBRAPHINData(
    dn_bundle,
    config=AtlasConfig(
        atlas_path="my_custom_atlas.nii.gz",
        roi_labels=["Region_A", "Region_B"],   # optional
    ),
).run()

Passing a NumPy array directly

import numpy as np

# Toy atlas: 4 cubic regions in an 8×8×8 volume
atlas_array = np.zeros((8, 8, 8), dtype=np.int16)
atlas_array[0:4, 0:4, 0:4] = 1
atlas_array[4:8, 0:4, 0:4] = 2
atlas_array[0:4, 4:8, 4:8] = 3
atlas_array[4:8, 4:8, 4:8] = 4

tx_bundle = TransformBRAPHINData(
    dn_bundle,
    atlas_data=atlas_array,
    config=AtlasConfig(),
).run()
print(tx_bundle.roi_time_series.shape)   # (4, T)

Stage 5 — Connectivity

Connectivity measures how similar two brain regions' activity patterns are over time. BRAPHIN supports 15 methods. The result is a square matrix (N_regions × N_regions) where each entry is the connectivity weight between a pair of regions.

Pearson correlation (recommended starting point)

from braphin import ModelBRAPHINConnectivityData, ConnectivityConfig

conn_bundle = ModelBRAPHINConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="pearson_correlation", threshold=0.3),
).run()

matrix = conn_bundle.connectivity_matrix   # ndarray (116, 116)
graph  = conn_bundle.graph                 # networkx.Graph

Cross-correlation

Measures similarity at the lag that maximises correlation, within a window of 10% of the signal length. The result matrix is not symmetric because the lag from region A to B may differ from B to A.

conn_bundle = ModelBRAPHINConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="cross_correlation"),
).run()

Granger causality (directed)

Tests whether the past activity of one region helps predict the future activity of another. The result matrix is asymmetric: entry [i, j] is the influence from region j to region i.

conn_bundle = ModelBRAPHINConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="granger_causality"),
).run()

Spectral methods (TR required)

conn_bundle = ModelBRAPHINConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="coherence"),
).run()

Thresholding

Set threshold to keep only edges with absolute weight at or above the threshold. Weaker connections are removed from the graph.

conn_bundle = ModelBRAPHINConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="pearson_correlation", threshold=0.5),
).run()

# List all available method keys programmatically
from braphin import list_fmri_connectivity_measures
print(list_fmri_connectivity_measures())
# ['pearson_correlation', 'partial_correlation', 'cross_correlation', ...]

Frequency Bands

BRAPHIN provides predefined BOLD fMRI frequency bands. These are different from EEG bands — BOLD oscillations are much slower. The bands are based on Zuo et al. (2010).

from braphin import (
    FMRI_BANDS,
    bandpass_roi_time_series,
    compute_band_connectivity,
    compute_all_bands_connectivity,
)

# FMRI_BANDS = {
#     "slow5":     (0.010, 0.027),   # Hz
#     "slow4":     (0.027, 0.073),
#     "slow3":     (0.073, 0.167),
#     "broadband": (0.010, 0.100),
# }

# Filter region signals to a single band
filtered = bandpass_roi_time_series(
    roi_time_series=tx_bundle.roi_time_series,
    tr=2.0,
    low=FMRI_BANDS["slow4"][0],
    high=FMRI_BANDS["slow4"][1],
)

# Compute connectivity for all four bands at once
band_results = compute_all_bands_connectivity(
    roi_time_series=tx_bundle.roi_time_series,
    tr=2.0,
    method="pearson_correlation",
)
# band_results is a dict: {"slow5": ndarray, "slow4": ndarray, "slow3": ndarray, "broadband": ndarray}

Unified EEG + fMRI API

BRAPHIN integrates with EEGraph through the Graph class. The same four-line pattern works for both modalities. Always import Graph from braphin, not from eegraph directly.

fMRI

from braphin import Graph, AtlasConfig

G = Graph()
G.load_data("sub-01_task-rest_bold.nii.gz", modality="fmri")

graph, matrix = G.modelate(
    window_size=None,
    connectivity="pearson_correlation",
    threshold=0.3,
    atlas_config=AtlasConfig(atlas_name="schaefer_100"),
)

EEG (unchanged from EEGraph)

from braphin import Graph

G = Graph()
G.load_data("sub-01_eeg.edf", modality="eeg")
graph, matrix = G.modelate(window_size=2, connectivity="pearson_correlation")

Error Handling

All BRAPHIN errors inherit from BRAPHINError. Use specific subclasses for fine-grained handling:

from braphin.exceptions import (
    BRAPHINError,
    BRAPHINInputError,
    BRAPHINFormatError,
    AtlasError,
    PreprocessingError,
    DenoisingError,
    TransformationError,
    ConnectivityError,
)

try:
    bundle = InputfMRIData("missing_file.nii.gz").load()
except BRAPHINInputError as e:
    print(f"Input problem: {e}")        # file not found / bad extension

try:
    conn = ModelBRAPHINConnectivityData(
        tx_bundle,
        ConnectivityConfig(method="unknown_method"),
    ).run()
except ConnectivityError as e:
    print(f"Connectivity error: {e}")   # unsupported method

# Catch all BRAPHIN errors at once
try:
    run_full_pipeline(path)
except BRAPHINError as e:
    print(f"Pipeline failed: {type(e).__name__}: {e}")

GNN-Ready Output

BRAPHIN's output is already in the format expected by PyTorch Geometric. The NetworkX graph has node attributes (brain region centroid positions in 3D space) and edge attributes (connectivity weights).

import torch
from torch_geometric.utils import from_networkx

G_nx = conn_bundle.graph   # networkx.Graph

# Convert to a PyTorch Geometric Data object
data = from_networkx(G_nx, group_node_attrs=["pos"], group_edge_attrs=["weight"])

print(data.x.shape)          # (116, 3) — node positions (world-space centroids)
print(data.edge_index.shape) # (2, num_edges)
print(data.edge_attr.shape)  # (num_edges, 1)
Requires: torch-geometric. Install with pip install ".[gnn]". See the Installation page for CUDA-specific notes.
