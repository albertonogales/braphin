User Guide

This guide walks through every stage of the BRAPHIN pipeline with complete, runnable code examples. By the end you will be able to go from a raw NIfTI file to a NetworkX functional-connectivity graph ready for graph neural network classification.

Pipeline Overview

BRAPHIN follows a linear staged pipeline. Each stage is a class that accepts a typed bundle from the previous stage and returns a new bundle. You can inspect, save, or branch from any bundle in the chain.

InputMRIData.load()          →  MRIInputBundle
    ↓
PreprocessMRIData.run()      →  MRIPreprocessBundle
    ↓
DenoiseMRIData.run()         →  MRIDenoiseBundle
    ↓
TransformMRIData.run()       →  MRITransformBundle
    ↓
ModelMRIConnectivityData.run() →  MRIConnectivityBundle

All classes and bundles are available from the top-level package:

from braphin import (
    InputMRIData, PreprocessMRIData, DenoiseMRIData,
    TransformMRIData, ModelMRIConnectivityData,
    InputConfig, PreprocessConfig, DenoiseConfig,
    AtlasConfig, ConnectivityConfig,
    MRIInputBundle, MRIPreprocessBundle, MRIDenoiseBundle,
    MRITransformBundle, MRIConnectivityBundle,
)

Stage 1 — Loading Data

InputMRIData validates and loads a 4-D NIfTI file (.nii or .nii.gz) along with any optional auxiliary files.

Basic load (no auxiliaries)

from braphin import InputMRIData

bundle = InputMRIData("sub-01_task-rest_bold.nii.gz").load()

print(bundle.fmri_path)              # path string
print(bundle.fmri_metadata["shape"]) # e.g. (91, 109, 91, 200)
print(bundle.fmri_metadata["ndim"])  # 4

Load with confound file

Pass a list of auxiliary paths. Supported extensions: .tsv, .csv, .npy, .json.

bundle = InputMRIData(
    "sub-01_task-rest_bold.nii.gz",
    auxiliary_paths=[
        "sub-01_task-rest_confounds_timeseries.tsv",
        "sub-01_task-rest_bold.json",           # optional sidecar
    ],
).load()

# Auxiliary data is in a dict keyed by filename
confounds_array = bundle.auxiliary_files["sub-01_task-rest_confounds_timeseries.tsv"]
print(confounds_array.shape)   # e.g. (200, 24)
BIDS TSV files: BRAPHIN automatically detects and skips the text header row in BIDS-format confound files (e.g. trans_x trans_y …). The returned array contains only numeric values — no NaN rows from the header.

Custom input configuration

from braphin import InputMRIData, InputConfig

cfg = InputConfig(
    allowed_fmri_extensions=[".nii", ".nii.gz"],
    allowed_aux_extensions=[".json", ".tsv", ".csv", ".npy"],
    allow_tabular_inputs=True,
)
bundle = InputMRIData("bold.nii.gz", config=cfg).load()

Stage 2 — Preprocessing

PreprocessMRIData applies basic data preparation to the raw 4-D array.

What it does

All six steps are implemented and execute in this fixed order:

  1. NaN / inf cleaning — always on. Non-finite values are replaced with 0.0.
  2. Slice-timing correction — linear interpolation per slice to a common reference time. Requires tr.
  3. Motion correction — rigid-body (6-parameter) realignment to volume 0 via Powell optimisation.
  4. Outlier detection / scrubbing — DVARS threshold; outlier volumes are interpolated from neighbours (or just marked).
  5. Global intensity normalisation — z-score over the whole 4-D volume.
  6. Spatial smoothing — isotropic Gaussian kernel; FWHM converted to per-axis sigma using NIfTI voxel sizes.

Example — all steps enabled

from braphin import PreprocessMRIData, PreprocessConfig

pp_bundle = PreprocessMRIData(
    input_bundle,
    PreprocessConfig(
        apply_slice_timing=True,      tr=2.0,            # TR in seconds
        apply_motion_correction=True,
        apply_outlier_detection=True, outlier_threshold_dvars=1.5,
        apply_normalization=True,
        apply_smoothing=True,         smoothing_fwhm=6.0,
    ),
).run()

print(pp_bundle.applied_steps)
# ['slice_timing_correction', 'motion_correction', 'outlier_detection',
#  'global_intensity_normalization', 'spatial_smoothing']
print(pp_bundle.motion_params.shape)    # (200, 6) — [tx,ty,tz,rx,ry,rz]
print(pp_bundle.outlier_mask.sum())     # number of flagged volumes

Minimal example — normalisation only

pp_bundle = PreprocessMRIData(
    input_bundle,
    PreprocessConfig(
        apply_slice_timing=False,
        apply_motion_correction=False,
        apply_outlier_detection=False,
        apply_normalization=True,
        apply_smoothing=False,
    ),
).run()
print(pp_bundle.applied_steps)    # ['global_intensity_normalization']

Inspecting metadata

meta = pp_bundle.preprocess_metadata
print(meta["original_shape"])           # (91, 109, 91, 200)
print(meta["num_voxels"])               # 907163
print(meta["num_timepoints"])           # 200
print(meta["non_finite_values_replaced"])  # 0 (if data is clean)

Stage 3 — Denoising

DenoiseMRIData regresses out nuisance confounds using ordinary least squares.

How confound regression works

Given a confound matrix C (T × K) and the voxel time-series matrix Y (T × V), BRAPHIN solves the normal equations and returns the residuals:

β = (CᵀC)⁻¹ Cᵀ Y       # via np.linalg.lstsq
Y_clean = Y − C β

Automatic confound file detection

The denoising stage searches the auxiliary files dictionary inherited from the input bundle. It tries each .tsv / .csv / .npy file and uses the first one whose number of rows matches the number of time points. Incompatible files (e.g. wrong number of rows) are silently skipped.

from braphin import DenoiseMRIData, DenoiseConfig

dn_bundle = DenoiseMRIData(
    pp_bundle,
    DenoiseConfig(regress_confounds=True),
).run()

print(dn_bundle.applied_steps)
# ['confound_regression']

print(dn_bundle.denoise_metadata["confounds_file_used"])
# 'sub-01_task-rest_confounds_timeseries.tsv'

print(dn_bundle.denoise_metadata["confounds_shape"])
# (200, 24)

Bandpass filtering

Zero-phase 5th-order Butterworth filter. Requires tr (seconds).

dn_bundle = DenoiseMRIData(
    pp_bundle,
    DenoiseConfig(
        regress_confounds=True,
        apply_bandpass=True,
        tr=2.0,                   # repetition time in seconds
        bandpass_low=0.008,       # Hz (default)
        bandpass_high=0.1,        # Hz (default — Biswal 1995 resting-state band)
    ),
).run()
print(dn_bundle.applied_steps)
# ['confound_regression', 'bandpass_filtering']

Scrubbing

Interpolates (or marks) outlier volumes detected by DVARS. When PreprocessMRIData has already run outlier detection, the stored mask is reused; otherwise DVARS is recomputed on the denoised signal.

dn_bundle = DenoiseMRIData(
    pp_bundle,
    DenoiseConfig(regress_confounds=True, apply_scrubbing=True),
).run()
print(dn_bundle.applied_steps)
# ['confound_regression', 'scrubbing']

Skipping denoising entirely

dn_bundle = DenoiseMRIData(
    pp_bundle,
    DenoiseConfig(regress_confounds=False, apply_scrubbing=False, apply_bandpass=False),
).run()
# applied_steps will be empty — data passes through unchanged

Stage 4 — Atlas Parcellation

TransformMRIData applies a brain atlas to the denoised fMRI and extracts one mean time series per ROI.

Using a bundled atlas

from braphin import TransformMRIData, AtlasConfig

tx_bundle = TransformMRIData(
    dn_bundle,
    config=AtlasConfig(atlas_name="aal"),  # 116 ROIs
).run()

print(tx_bundle.roi_time_series.shape)     # (116, 200)
print(tx_bundle.roi_labels[:3])            # ['Precentral_L', 'Precentral_R', ...]
print(tx_bundle.centroid_coordinate_space) # 'world'

Automatic atlas resampling

If the atlas and fMRI have different voxel sizes or field-of-view, BRAPHIN automatically resamples the atlas to the fMRI space using nearest-neighbour interpolation (via nibabel resample_from_to with order=0). Label integrity is preserved.

Using a custom atlas path

tx_bundle = TransformMRIData(
    dn_bundle,
    config=AtlasConfig(
        atlas_path="my_custom_atlas.nii.gz",
        roi_labels=["Region_A", "Region_B", ...],  # optional
    ),
).run()

Passing a NumPy array or NIfTI image directly

import numpy as np
import nibabel as nib

# Build a toy atlas: 4 cubic ROIs in an 8×8×8 volume
atlas_array = np.zeros((8, 8, 8), dtype=np.int16)
atlas_array[0:4, 0:4, 0:4] = 1
atlas_array[4:8, 0:4, 0:4] = 2
atlas_array[0:4, 4:8, 4:8] = 3
atlas_array[4:8, 4:8, 4:8] = 4

tx_bundle = TransformMRIData(
    dn_bundle,
    atlas_data=atlas_array,
    config=AtlasConfig(),      # atlas_name=None
).run()

print(tx_bundle.roi_time_series.shape)      # (4, T)
print(tx_bundle.centroid_coordinate_space)  # 'voxel' (no affine available)

ROI centroid caching

For bundled atlases, world-space centroids are pre-computed and stored as JSON files in braphin/atlas_centroids/. The cache is used automatically and avoids recomputing centroids on every run. Custom atlases compute centroids from the data at runtime.

Stage 5 — Connectivity

ModelMRIConnectivityData computes a ROI × ROI connectivity matrix and builds a NetworkX graph.

Pearson correlation

from braphin import ModelMRIConnectivityData, ConnectivityConfig

conn_bundle = ModelMRIConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="pearson_correlation", threshold=0.3),
).run()

matrix = conn_bundle.connectivity_matrix   # ndarray (116, 116)
graph  = conn_bundle.graph                 # networkx.Graph

Cross-correlation

Normalised cross-correlation at the lag that maximises |r| within a window of 10% of the signal length:

conn_bundle = ModelMRIConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="cross_correlation"),
).run()

Corrected cross-correlation

Antisymmetric measure: corCC(x, y) = R_xy(+lag) − R_xy(−lag). Satisfies corCC(x, y) = −corCC(y, x). The resulting matrix is not symmetric:

conn_bundle = ModelMRIConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="corr_cross_correlation"),
).run()

Threshold

Set threshold to keep only edges with absolute weight ≥ threshold:

conn_bundle = ModelMRIConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="pearson_correlation", threshold=0.5),
).run()
# Edges with |r| < 0.5 are removed from the graph

All connectivity methods

Method keyDescriptionSymmetric
pearson_correlationPearson r between ROI time series✅ Yes
cross_correlationNormalised cross-correlation (max-lag)❌ No
corr_cross_correlationCorrected cross-correlation (antisymmetric)❌ No (antisymmetric)
 
# Programmatically list all available methods
from braphin import list_connectivity_measures
print(list_connectivity_measures())
# ['pearson_correlation', 'cross_correlation', 'corr_cross_correlation']

Unified EEG + fMRI API

BRAPHIN integrates with EEGraph through the same Graph interface used for EEG data:

fMRI via unified API

from eegraph.graph import Graph
from braphin import AtlasConfig

G = Graph()
G.load_data("sub-01_task-rest_bold.nii.gz", modality="fmri")

graph, matrix = G.modelate(
    window_size=None,
    connectivity="pearson_correlation",
    threshold=0.3,
    atlas_config=AtlasConfig(atlas_name="schaefer_100"),
)

EEG via unified API (unchanged from EEGraph)

G = Graph()
G.load_data("sub-01_eeg.edf", modality="eeg")
graph, matrix = G.modelate(window_size=2, connectivity="pearson_correlation")

Error Handling

All BRAPHIN errors inherit from BRAPHINError. Use specific subclasses for fine-grained handling:

from braphin.exceptions import (
    BRAPHINError,
    MRIInputError,
    MRIFormatError,
    AtlasError,
    PreprocessingError,
    DenoisingError,
    TransformationError,
    ConnectivityError,
)

try:
    bundle = InputMRIData("missing_file.nii.gz").load()
except MRIInputError as e:
    print(f"Input problem: {e}")        # file not found / bad extension

try:
    conn = ModelMRIConnectivityData(
        tx_bundle,
        ConnectivityConfig(method="unknown_method"),
    ).run()
except ConnectivityError as e:
    print(f"Connectivity error: {e}")   # unsupported method

# Catch all BRAPHIN errors at once
try:
    run_full_pipeline(path)
except BRAPHINError as e:
    print(f"Pipeline failed: {type(e).__name__}: {e}")

Exception hierarchy

ExceptionRaised when
MRIInputErrorFile not found, unsupported extension
MRIFormatErrorWrong NIfTI dimensionality, unexpected shape
AtlasErrorUnknown atlas name, shape mismatch between atlas and fMRI
PreprocessingErrorInvalid bundle, zero-variance signal (cannot normalise)
DenoisingErrorNo compatible confound file found, regression failure
TransformationErrorAtlas–fMRI space mismatch, empty ROIs
ConnectivityErrorUnsupported method, matrix computation failure

GNN-Ready Output

The connectivity bundle's NetworkX graph has node attributes (centroid positions) and edge attributes (connectivity weights). Converting to PyTorch Geometric:

import torch
from torch_geometric.utils import from_networkx

G_nx = conn_bundle.graph   # networkx.Graph

# Convert to PyTorch Geometric Data object
data = from_networkx(G_nx, group_node_attrs=["pos"], group_edge_attrs=["weight"])

print(data.x.shape)          # (116, 3) — node positions (world-space centroids)
print(data.edge_index.shape) # (2, num_edges)
print(data.edge_attr.shape)  # (num_edges, 1)
Requires: torch-geometric. Install with pip install ".[gnn]". See the Installation page for CUDA-specific notes.

See the Examples/ folder in the repository for complete Jupyter notebooks demonstrating Parkinson's disease classification using GNNs built on BRAPHIN connectivity graphs.
