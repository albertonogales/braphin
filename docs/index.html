🧠 BRAPHIN

A Python library for resting-state fMRI functional connectivity analysis. From NIfTI file to NetworkX graph in five pipeline stages.

Python ≥ 3.9 167 Tests Passing MIT License v1.0.0

Features

🔬

Full fMRI Pipeline

NIfTI loading, preprocessing, confound regression, atlas parcellation, ROI extraction and connectivity — all in one coherent API.

🗺️

Four Bundled Atlases

AAL (116 ROIs), Schaefer 100, 200, and 400. Automatic nearest-neighbour resampling to the subject's fMRI space.

📊

Three Connectivity Measures

Pearson correlation, cross-correlation, and corrected cross-correlation with optional absolute threshold.

🕸️

GNN-Ready Output

NetworkX graphs with node positions and edge weights. Compatible with PyTorch Geometric for GNN-based classification tasks.

🔗

Unified EEG + fMRI API

Extends EEGraph. Use the same Graph.load_data() / Graph.modelate() calls for both modalities.

🧪

Fully Tested

167 pytest unit and integration tests. All tests use synthetic data — no external files required.

Pipeline Overview

Every stage accepts a typed bundle dataclass from the previous stage and produces a new bundle. This ensures full traceability and makes it easy to inspect, checkpoint, or restart at any stage.

1
InputMRIData
Validates and loads the NIfTI file. Accepts auxiliary files (TSV confounds, CSV, NPY, JSON). Returns MRIInputBundle.
▼
2
PreprocessMRIData
Cleans NaN/inf values; optional slice-timing correction, motion correction, outlier detection, per-voxel temporal z-score normalisation, and spatial smoothing. Returns MRIPreprocessBundle.
▼
3
DenoiseMRIData
Least-squares confound regression using motion parameters or any TSV/CSV confound file. Returns MRIDenoiseBundle.
▼
4
TransformMRIData
Atlas parcellation with automatic resampling. Extracts per-ROI mean time series and world-space centroids. Returns MRITransformBundle.
▼
5
ModelMRIConnectivityData
Computes the ROI×ROI connectivity matrix and builds a NetworkX graph. Returns MRIConnectivityBundle.

Quick Start

from braphin import (
    InputMRIData, PreprocessMRIData, DenoiseMRIData,
    TransformMRIData, ModelMRIConnectivityData,
    PreprocessConfig, DenoiseConfig, AtlasConfig, ConnectivityConfig,
)

# 1 — Load
input_bundle = InputMRIData(
    "sub-01_task-rest_bold.nii.gz",
    auxiliary_paths=["sub-01_task-rest_confounds_timeseries.tsv"],
).load()

# 2 — Preprocess
pp_bundle = PreprocessMRIData(
    input_bundle,
    PreprocessConfig(apply_normalization=True),
).run()

# 3 — Denoise: regress out motion confounds
dn_bundle = DenoiseMRIData(
    pp_bundle,
    DenoiseConfig(regress_confounds=True),
).run()

# 4 — Parcellate with the AAL atlas (116 ROIs)
tx_bundle = TransformMRIData(
    dn_bundle,
    config=AtlasConfig(atlas_name="aal"),
).run()

# 5 — Compute Pearson functional connectivity
conn_bundle = ModelMRIConnectivityData(
    tx_bundle,
    ConnectivityConfig(method="pearson_correlation", threshold=0.3),
).run()

matrix = conn_bundle.connectivity_matrix   # ndarray shape (116, 116)

Design Principles

  • Bundle pattern: each stage consumes and produces a typed dataclass — full pipeline traceability with no hidden state.
  • Strategy pattern: connectivity measures are interchangeable without touching pipeline code.
  • Fail loudly: a seven-class exception hierarchy (BRAPHINError → stage-specific) with actionable messages — no silent failures.
  • Synthetic tests only: the full test suite runs without any real neuroimaging data files.
