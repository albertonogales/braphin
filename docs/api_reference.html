Configuration Dataclasses

All configuration objects live in braphin.config and are re-exported from the top-level package.

dataclass InputConfig

Controls NIfTI loading behaviour.

FieldTypeDefaultDescription
allowed_fmri_extensionslist[str][".nii", ".nii.gz"]Accepted file extensions for the main fMRI file
allowed_aux_extensionslist[str][".json", ".tsv", ".csv", ".npy"]Accepted file extensions for auxiliary files
allow_tabular_inputsboolTrueWhether tabular auxiliary inputs are accepted
dataclass PreprocessConfig

Controls the preprocessing stage. All six steps are fully implemented and execute in order: NaN cleaning → slice timing → motion correction → outlier detection → per-voxel z-score normalisation → smoothing.

FieldTypeDefaultDescription
apply_motion_correctionboolFalse✅ Rigid-body realignment to volume 0 (Powell optimisation)
apply_slice_timingboolFalse✅ Linear interpolation per-slice to reference time. Requires tr.
apply_outlier_detectionboolFalse✅ DVARS-threshold outlier detection + scrubbing
apply_voxel_zscoreboolTrue✅ Per-voxel temporal z-score normalisation (mean and std computed along the time axis)
apply_smoothingboolFalse✅ Isotropic Gaussian spatial smoothing
smoothing_fwhmfloat6.0Full-width at half maximum of the Gaussian kernel, in mm
trfloat | NoneNoneRepetition time in seconds. Required when apply_slice_timing=True
slice_orderstr"sequential"Slice acquisition order: "sequential" or "interleaved"
slice_timing_ref_sliceint0Reference slice index for STC (0 = first, -1 = middle)
slice_axisint2Spatial axis corresponding to the slice dimension: 0 = X (sagittal), 1 = Y (coronal), 2 = Z (axial). Change for non-axial acquisitions. Must be 0, 1, or 2.
outlier_threshold_dvarsfloat1.5IQR multiplier for DVARS outlier threshold
scrubbing_strategystr"interpolate""interpolate": replace outliers with neighbour interpolation; "mark": flag only
dataclass DenoiseConfig

Controls the denoising stage. All three steps are fully implemented. When both regress_confounds=True and apply_bandpass=True, confounds are bandpass-filtered with the same filter before OLS regression (Lindquist et al. 2019 order: scrubbing → bandpass → regression).

FieldTypeDefaultDescription
regress_confoundsboolTrue✅ Ordinary least-squares confound regression
apply_scrubbingboolTrue✅ Interpolates outlier volumes (uses preprocess outlier_mask if available, else recomputes DVARS)
apply_bandpassboolTrue✅ Zero-phase 5th-order Butterworth bandpass filter. Requires tr.
trfloat | NoneNoneRepetition time in seconds. Required when apply_bandpass=True
bandpass_lowfloat0.008Low cut-off frequency in Hz
bandpass_highfloat0.1High cut-off frequency in Hz — Biswal 1995 resting-state band upper bound (must be < Nyquist = 0.5/TR)
dataclass AtlasConfig

Selects which atlas to use in TransformBRAPHINData. Exactly one of atlas_name or atlas_path should be set (or neither when passing atlas_data directly to the constructor).

FieldTypeDefaultDescription
atlas_namestr | NoneNoneOne of "aal", "schaefer_100", "schaefer_200", "schaefer_400"
atlas_pathstr | NoneNonePath to a custom NIfTI atlas file
roi_labelslist[str] | NoneNoneOptional list of custom ROI names (length must equal number of unique atlas labels)
dataclass ConnectivityConfig

Controls the connectivity computation stage.

FieldTypeDefaultDescription
methodstr"pearson_correlation"Connectivity measure key. See Connectivity tools for all 15 supported methods.
window_sizefloat | NoneNoneNone = static connectivity. A positive float (seconds) requests windowed dynamic connectivity (planned; flagged as pending step).
thresholdfloat | NoneNoneIf set, edges with |weight| < threshold are removed from the graph
dataclass BRAPHINConfig

Aggregate configuration for the entire pipeline. Bundles all stage-specific configs into a single object for convenience.

FieldTypeDefault
input_configInputConfigInputConfig()
preprocess_configPreprocessConfigPreprocessConfig()
denoise_configDenoiseConfigDenoiseConfig()

Bundle Dataclasses

Each pipeline stage returns a bundle — a typed dataclass that carries data, metadata, and provenance information to the next stage. All bundles are frozen-safe: they can be serialised (e.g. pickled) for checkpointing.

dataclass BRAPHINInputBundle

Output of InputfMRIData.load().

FieldTypeDescription
fmri_pathstr | NoneAbsolute path to the source NIfTI file
fmri_imagenibabel.Nifti1Image | NoneLoaded NIfTI image object
fmri_metadatadict | NoneKeys: shape, ndim, affine, header, tr (auto-extracted from pixdim[4]; BIDS sidecar "RepetitionTime" takes precedence), slice_timing_offsets (from BIDS sidecar "SliceTiming", if present)
auxiliary_filesdict[str, ndarray]Filename → loaded array for each auxiliary file
dataclass BRAPHINPreprocessBundle

Output of PreprocessBRAPHINData.run().

FieldTypeDescription
fmri_pathstr | NoneInherited from input bundle
original_metadatadict | NoneMetadata from the input bundle
preprocessed_datandarray (X, Y, Z, T)4-D float32 array after cleaning and optional normalisation
voxel_time_seriesndarray (V, T)Reshaped 2-D view: V = X·Y·Z voxels, T = time points
auxiliary_filesdictInherited from input bundle
applied_stepslist[str]Steps actually executed (e.g. "slice_timing_correction", "motion_correction", "per_voxel_temporal_normalisation", …)
pending_stepslist[str]Always [] — retained for backward compatibility; all preprocessing steps are fully implemented
motion_paramsndarray (T, 6) | NoneEstimated rigid-body motion parameters [tx, ty, tz, rx, ry, rz] per volume. None if motion correction was not applied.
outlier_maskndarray (T,) bool | NoneBoolean mask marking outlier volumes detected via DVARS. None if outlier detection was not applied.
preprocess_metadatadictKeys: original_shape, num_voxels, num_timepoints, non_finite_values_replaced, motion_params_shape, n_outliers, …

atlas_namestr | NoneName of the bundled atlas used, or None for custom

Pipeline Classes

class InputfMRIData

Stage 1 — validates and loads a NIfTI fMRI file and optional auxiliary files.

InputfMRIData(fmri_path: str | Path, auxiliary_paths: list | None = None, config: InputConfig | None = None)
ParameterDescription
fmri_pathPath to the .nii or .nii.gz file
auxiliary_pathsOptional list of confound / event / JSON sidecar paths
configOptional InputConfig; defaults to InputConfig()
load() → BRAPHINInputBundle

Executes validation and loading. Raises BRAPHINInputError for missing files or bad extensions; BRAPHINFormatError if the NIfTI is not 4-D.

display_info(bundle: BRAPHINInputBundle) → None

Prints a human-readable summary of the loaded bundle to stdout. Output is prefixed with [BRAPHIN].

class PreprocessBRAPHINData

Stage 2 — NaN/inf cleaning and optional preprocessing steps including slice-timing correction, motion correction, outlier detection, per-voxel temporal z-score normalisation, and spatial smoothing.

PreprocessBRAPHINData(input_bundle: BRAPHINInputBundle, config: PreprocessConfig | None = None)
run() → BRAPHINPreprocessBundle

Executes preprocessing. Raises PreprocessingError if the bundle is invalid or if normalisation fails (zero-variance signal).

function get_motion_confounds(preprocess_bundle)

get_motion_confounds(preprocess_bundle)

Return motion parameters suitable for confound regression (negated). See preprocess section.

Returns: ndarray (T, 6) — negated world-space motion parameters.

The stored motion_params represent the forward rigid-body transform mapping the moving volume to the reference. Confound regression requires the negated (inverse) parameters. Raises PreprocessingError if motion correction was not applied.

class DenoiseBRAPHINData

Stage 3 — least-squares confound regression, optional scrubbing, and optional bandpass filtering.

DenoiseBRAPHINData(preprocess_bundle: BRAPHINPreprocessBundle, config: DenoiseConfig | None = None)
run() → BRAPHINDenoiseBundle

Scans auxiliary files for a compatible confound matrix (rows = time points) and regresses it out. Incompatible files are skipped. Raises DenoisingError if regress_confounds=True but no compatible file is found.

class TransformBRAPHINData

Stage 4 — atlas parcellation, automatic resampling, ROI mean time-series extraction and centroid computation.

TransformBRAPHINData(denoise_bundle: BRAPHINDenoiseBundle, config: AtlasConfig | None = None, atlas_data: ndarray | Nifti1Image | None = None)
ParameterDescription
denoise_bundleOutput of DenoiseBRAPHINData.run()
configAtlas selection. Use AtlasConfig(atlas_name="aal") for bundled atlases
atlas_dataAlternative to config.atlas_path: pass a NumPy array or NIfTI image directly. When set, config.atlas_name should be None
run() → BRAPHINTransformBundle

Resamples the atlas to fMRI space if needed, extracts ROI mean time series, computes centroids, and writes a centroid JSON cache for bundled atlases. Raises AtlasError or TransformationError on failure.

class ModelBRAPHINConnectivityData

Stage 5 — computes the connectivity matrix and builds a NetworkX graph.

ModelBRAPHINConnectivityData(transform_bundle: BRAPHINTransformBundle, config: ConnectivityConfig | None = None)
run() → BRAPHINConnectivityBundle

Computes the N×N connectivity matrix using the configured method, optionally applies the absolute threshold, and builds a NetworkX graph with node positions and edge weights. Raises ConnectivityError for unsupported methods or matrix failures.

BRAPHINGraph (exported as Graph)

Available as from braphin import Graph or from braphin import BRAPHINGraph. Subclasses eegraph.graph.Graph. The dependency is strictly one-way: braphin imports from eegraph; eegraph has zero imports from braphin.

class BRAPHINGraph

Unified entry point for both fMRI and EEG connectivity analyses.

load_data(path: str, modality: str = "fmri", auxiliary_paths: list | None = None) → None

Loads data for the specified modality. modality="fmri" routes to the BRAPHIN pipeline; modality="eeg" routes to the EEGraph pipeline.

modelate(window_size: float | None, connectivity: str, threshold: float | None = None, atlas_config: AtlasConfig | None = None, **kwargs) → tuple[networkx.Graph, ndarray]

Runs the full connectivity pipeline for the loaded modality and returns (networkx_graph, connectivity_matrix). For fMRI, atlas_config selects the parcellation atlas; for EEG, window_size controls windowing.

ModelMRIData

Available from braphin. High-level orchestrator that runs the full fMRI pipeline in one call.

class ModelMRIData

Wraps all five pipeline stages (Input → Preprocess → Denoise → Transform → Connectivity) behind a single method.

ModelMRIData(fmri_path: str, auxiliary_paths: list | None = None, config: BRAPHINConfig | None = None)
connectivity_workflow() → BRAPHINConnectivityBundle

Executes the full pipeline from NIfTI file to connectivity bundle. Uses the configs provided at construction time. Returns the final BRAPHINConnectivityBundle.

Frequency Bands (braphin.bands)

Available from braphin.bands and re-exported from the top-level package. These are BOLD fMRI oscillation bands; they are not EEG bands.

constant FMRI_BANDS: dict[str, tuple[float, float]]

Predefined BOLD fMRI frequency bands based on Zuo et al. (2010):

KeyRange (Hz)Description
slow50.010 – 0.027Slow-5 BOLD band
slow40.027 – 0.073Slow-4 BOLD band
slow30.073 – 0.167Slow-3 BOLD band
broadband0.010 – 0.100Broadband resting-state band
function bandpass_roi_time_series(roi_time_series, tr, low, high) → ndarray

Applies a zero-phase Butterworth bandpass filter to an (N_rois, T) ROI time-series matrix. Returns the filtered matrix with the same shape.

function compute_band_connectivity(roi_time_series, tr, band_name, method) → ndarray

Bandpass-filters the ROI time series to the named BOLD band, then computes connectivity using the specified method. band_name must be a key of FMRI_BANDS. Returns an (N, N) connectivity matrix.

function compute_all_bands_connectivity(roi_time_series, tr, method) → dict[str, ndarray]

Runs compute_band_connectivity for all four BOLD bands and returns a dict mapping band name to connectivity matrix.

Available from braphin.atlas.

Immutable descriptor for a supported atlas.

FieldTypeDescription

num_roisintNumber of regions
modalitystrIntended modality (e.g. "fMRI")

function list_supported_atlases() → list[str]

Returns the list of all registered atlas names: ["aal", "schaefer_100", "schaefer_200", "schaefer_400"].

function get_atlas_definition(atlas_name: str) → AtlasDefinition

Returns the AtlasDefinition for the given name. Raises AtlasError if the name is not registered.

function get_atlas_roi_name_map(atlas_name: str) → dict[int, str]

Returns a mapping from integer NIfTI label (1-based) to anatomical ROI name. For AAL, keys are 1–116 (matching the sequential NIfTI labels, not the coded anatomical IDs 2001, 2101, …). Returns an empty dict for atlases without a name map (Schaefer).

Strategy Pattern

Available from braphin.strategy. You can subclass ConnectivityStrategy to implement custom measures without modifying pipeline code.

abstract class ConnectivityStrategy

ABC for all connectivity strategies.

compute(roi_time_series: ndarray) → ndarray

Must return an N × N connectivity matrix from an (N, T) input.

function get_connectivity_strategy(method: str) → ConnectivityStrategy

Factory function. Returns the concrete strategy instance for method. Raises ConnectivityError for unknown method strings.

function validate_connectivity_method(method: str) → None

Raises ConnectivityError if method is not in CONNECTIVITY_MEASURES.

function compute_pearson_correlation(roi_time_series: ndarray) → ndarray

Computes the N×N Pearson correlation matrix from an (N, T) ROI time-series array using numpy.corrcoef. Diagonal is 1.0; off-diagonal values are in [−1, 1].

function compute_partial_correlation(roi_time_series: ndarray) → ndarray

Computes the N×N partial correlation matrix via the precision matrix (inverse of the covariance matrix). Diagonal is 1.0; values are clamped to [−1, 1].

Note: Raises ConnectivityError if the number of ROIs is greater than or equal to the number of timepoints (rank-deficient covariance matrix). Consider using Pearson correlation or reducing the number of ROIs in that case.

function apply_connectivity_threshold(matrix: ndarray, threshold: float) → ndarray

Returns a copy of matrix where entries with |value| < threshold are set to 0. Does not modify the diagonal.

Exception Hierarchy

All exceptions are in braphin.exceptions and are importable from there.

exception BRAPHINError(Exception)

Base class for all BRAPHIN errors. Catch this to handle any pipeline failure generically.

exception BRAPHINInputError(BRAPHINError)

Raised by InputfMRIData. File not found, unsupported extension, missing auxiliary file.

exception BRAPHINFormatError(BRAPHINError)

Raised when the NIfTI file has unexpected dimensionality (not 4-D) or internal format issues.

exception AtlasError(BRAPHINError)

Unknown atlas name, missing atlas file, shape mismatch between atlas and fMRI volume.

exception PreprocessingError(BRAPHINError)

Invalid input bundle, zero-variance signal preventing normalisation.

exception DenoisingError(BRAPHINError)

No compatible confound file found when regress_confounds=True; confound regression failure.

exception TransformationError(BRAPHINError)

Atlas–fMRI space mismatch after resampling; empty ROI mask; centroid computation failure.

exception ConnectivityError(BRAPHINError)

Unsupported connectivity method string; ROI time-series matrix has wrong shape; matrix computation failure.
